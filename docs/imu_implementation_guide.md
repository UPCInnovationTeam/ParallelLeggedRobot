# IMU FreeRTOS轮询模式实现指南

## 概述

本项目实现了一个基于FreeRTOS的IMU数据采集系统，运行在ESP32的Core1上，使用互补滤波器进行传感器融合，计算欧拉角（俯仰、横滚、偏航）。

## 架构设计

### 系统架构图

```
[ICM20602传感器] 
        ↓
[IMU类] - 读取原始数据并转换为物理单位
        ↓
[互补滤波器] - 融合加速度计和陀螺仪数据
        ↓
[FreeRTOS任务 (Core1, 200Hz)] - 轮询模式采集数据
        ↓
[数据队列] - 传递处理后的IMU数据
        ↓
[主程序] - 使用数据进行平衡控制
```

### 关键组件

#### 1. IMU类 (`imu.h` / `imu.cpp`)
- **数据结构**:
  - `AccelData_t`: 加速度计数据 (m/s²)
  - `GyroData_t`: 陀螺仪数据 (rad/s)
  - `IMU_Data_t`: 处理后的欧拉角数据

- **主要方法**:
  - `readAccelData() / readGyroData()`: 读取原始传感器数据
  - `convertToPhysicalUnits()`: 转换为物理单位
  - `calculateAccelerometerAngles()`: 计算加速度计倾斜角
  - `integrateGyroData()`: 积分陀螺仪数据
  - `applyComplementaryFilter()`: 应用互补滤波器
  - `updateSensorFusion()`: 完整的传感器融合流程

#### 2. FreeRTOS IMU任务 (`imu_task.cpp`)
- **任务参数**:
  - 优先级: 3 (高优先级)
  - 核心Core1 (时间关键任务)
  - 栈大小: 4096字节
  - 采样频率: 200Hz (5ms周期)

- **数据队列**:
  - 长度: 10
  - 非阻塞写入（覆盖旧数据）
  - 非阻塞读取

#### 3. 主程序 (`main.cpp`)
- 初始化FreeRTOS IMU任务
- 通过队列获取IMU数据
- 调用平衡控制逻辑

## 互补滤波器原理

### 公式
```
filtered_angle = α × (gyro_angle + gyro_rate × dt) + (1 - α) × accel_angle
```

其中:
- `α` = 0.98 (互补滤波器系数)
- `gyro_angle`: 陀螺仪积分得到的角度
- `accel_angle`: 加速度计计算的角度
- `dt`: 采样时间间隔

### 参数调整
- **α值增大**: 更多依赖陀螺仪，响应更快但漂移更明显
- **α值减小**: 更多依赖加速度计，更稳定但响应较慢
- 推荐: 0.95-0.99之间

## 性能优化建议

1. **采样频率**:
   - 根据需求调整，200Hz适合平衡控制
   - 避免超过300Hz（I2C限制）

2. **队列大小**:
   - 10个足够，过大会浪费内存
   - 采用"覆盖"策略确保数据新鲜度

3. **任务优先级**:
   - IMU任务优先级3，高于控制任务2
   - 确保数据采集不被阻塞

4. **内存管理**:
   - 注意栈大小设置
   - 监控堆内存使用情况

## 故障排查

### 常见问题

1. **队列满错误**
   - 现象: 数据丢失
   - 解决: 增加队列长度或降低采样频率

2. **采样频率不稳定**
   - 现象: 频率波动大
   - 解决: 检查任务优先级，确保没有更高优先级任务阻塞

3. **角度漂移**
   - 现象: 静止时角度缓慢变化
   - 解决: 调整互补滤波器系数，增加零漂补偿

4. **响应延迟**
   - 现象: 角度变化延迟明显
   - 解决: 降低α值，增加对加速度计的依赖

### 调试技巧

1. 使用 `printRawData()` 和 `printProcessedData()` 对比原始和处理后的数据
2. 监控任务执行时间，确保不超过5ms
3. 检查I2C通信错误（如果有的话）

## 扩展功能

### 1. 卡尔曼滤波器
如果需要更高精度的滤波，可以考虑实现卡尔曼滤波器替代互补滤波器。

### 2. DMP集成
ICM20602支持DMP（数字运动处理器），可以硬件加速姿态解算。

### 3. 多传感器融合
可以加入磁力计获取绝对航向，实现更完整的姿态估计。
